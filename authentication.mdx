## Overview

Secure your AlisX account using multiple authentication methods. AlisX supports API keys for server-to-server communication, OAuth 2.0 for third-party integrations, and role-based access control (RBAC) for team permissions. Enable multi-factor authentication (MFA) to add an extra layer of protection. Follow these guides to implement authentication correctly.

<Callout kind="alert">
Store your credentials securely. Never expose API keys or tokens in client-side code or public repositories.
</Callout>

## Authentication Methods

Choose the method that fits your use case. Use API keys for simple backend access and OAuth for user-facing applications.

## Generate an API Key

Navigate to your AlisX dashboard at `https://dashboard.example.com/settings/api-keys` and create a new key.

<Steps>
  <Step title="Create Key" icon="plus">
    Click **New API Key** and select the required scopes.
  </Step>
  <Step title="Copy Key" icon="copy">
    Copy the generated key immediately—it displays only once.
  </Step>
  <Step title="Store Securely" icon="lock">
    Save it in your environment variables as `ALISX_API_KEY`.
  </Step>
</Steps>

### Use the API Key

Include the key in the `Authorization` header.

<ParamField header="Authorization" param-type="string" required="true">
  Format: `Bearer YOUR_API_KEY`
</ParamField>

<CodeGroup tabs="JavaScript,Python,cURL">
  ```javascript
  const response = await fetch('https://api.example.com/v1/projects', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    }
  });
  ```
  ```python
  import requests

  headers = {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
  }
  response = requests.get('https://api.example.com/v1/projects', headers=headers)
  ```
  ```bash
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       -H "Content-Type: application/json" \
       https://api.example.com/v1/projects
  ```
</CodeGroup>

## OAuth Flow

AlisX uses OAuth 2.0 with PKCE for secure third-party access. Redirect users to `https://auth.example.com/oauth/authorize`.

<ParamField query="client_id" param-type="string" required="true">
  Your OAuth client ID from the dashboard.
</ParamField>

<ParamField query="redirect_uri" param-type="string" required="true">
  Registered callback URL.
</ParamField>

<ParamField query="scope" param-type="string" required="false">
  Space-separated scopes, e.g., `read:projects write:deploy`.
</ParamField>

### Authorization URL Example

```
https://auth.example.com/oauth/authorize?client_id=YOUR_CLIENT_ID&redirect_uri=https://yourapp.com/callback&scope=read:projects&response_type=code
```

Exchange the code for a token at `https://auth.example.com/oauth/token`.

```json
  "expires_in": 3600,
  "refresh_token": "def50200..."
}
```

## Role-Based Access Control (RBAC)

AlisX uses RBAC to manage permissions. Assign roles like Admin, Developer, or Viewer to team members.

| Role     | Permissions                          |
|----------|--------------------------------------|
| Admin    | Full access: manage users, projects, billing |
| Developer| Create/deploy projects, view billing |
| Viewer   | Read-only access to projects         |

<Expandable title="Advanced RBAC Configuration" default-open="false">
Customize permissions via the API.

<Request show-lines="true">
```javascript
await fetch('https://api.example.com/v1/teams/TEAM_ID/roles', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Custom Role',
    permissions: ['project:read', 'deploy:write']
  })
});
```
</Request>
</Expandable>

## Multi-Factor Authentication Setup

Enable MFA for all users to prevent unauthorized access.

<Steps>
  <Step title="Enable MFA" icon="settings">
    Go to `https://dashboard.example.com/security/mfa` and click **Enable**.
  </Step>
  <Step title="Scan QR Code" icon="smartphone">
    Use your authenticator app (e.g., Google Authenticator) to scan the QR code.
  </Step>
  <Step title="Verify" icon="check-circle">
    Enter the 6-digit code to confirm setup.
  </Step>
</Steps>

<Callout kind="tip">
Require MFA for all team members in your organization settings.
</Callout>

## Security Best Practices

- Rotate API keys every 90 days.
- Use least-privilege scopes for OAuth applications.
- Monitor logs at `https://dashboard.example.com/logs` for suspicious activity.
- Never commit secrets to version control—use environment variables.
